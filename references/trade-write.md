# 交易记录录入

`trade_calc.py --write-trade` — 用户发送交易截图（卖出/买入确认单），写入交易表并实时核算持仓表。

## 触发场景

- 用户说「新增交易」「录入交易」「记一笔交易」「有一笔卖出/买入」
- 用户发送了**交易截图/确认单**（同花顺/支付宝/天天基金等）

> **注意**：定投截图走 [dca-update.md](dca-update.md) 而非本流程。

## 工作流程

1. 读取交易表该标的全量历史交易记录
2. 以**交易表**为唯一数据源，计算历史买入/卖出汇总
3. 写入新交易记录（份额/金额/成本按方向自动正负）
4. 以**交易表汇总**重算持仓：总份额、总成本、最新市值、持有收益/收益率、年化收益率
5. 实时更新持仓表

## 命令

```bash
# 截图识别（统一使用 --force-precision 精准 API 模式）
mineru2md --file {图片路径} --output /tmp/ --force-precision

# 方式一：一体化（写入交易 + 更新持仓，一步完成）
python scripts/trade_calc.py \
  --code 007994 \
  --name "华夏中证500指数增强A" \
  --direction sell \
  --date "2026-05-28 14:58:03" \
  --shares 3500 \
  --nav 2.665 \
  --amount 9330.00 \
  --write-trade

# 方式二：先预览历史，不写入
python scripts/trade_calc.py --code 007994 --list

# 方式三：仅重新核算持仓，不写入交易
python scripts/trade_calc.py --code 007994 --recalc
```

## 卖出记录格式规范

| 字段 | 规则 | 示例 |
|------|------|------|
| 份额 | 负数 | -1588.96 |
| 金额 | 负数 | -4283.84 |
| 成本 | 负数（平均成本法） | -2620.04 |
| 收益 | 正=盈利，负=亏损 | 1663.80 |
| 收益率 | 百分比格式，带 % | 63.50% |
| 交易日期 | 格式 `YYYY-MM-DD HH:MM:SS` | 2026-05-28 14:59:23 |

## 平均成本法卖出计算

```
卖出成本 = 卖出份额 × (该标的历史买入总成本 / 历史买入总份额)
卖出收益 = |卖出金额| - |卖出成本|
卖出收益率 = 卖出收益 / |卖出成本| × 100%
```

## 持仓表重算规则（卖出后）

1. **总份额** = 历史买入总份额 - 历史卖出总份额
2. **总成本** = 历史买入总成本 - 历史卖出总成本（平均成本法）
3. **市值** = 总份额 × 最新净值
4. **持有收益** = 市值 - 总成本
5. **持有收益率** = 持有收益 / 总成本 × 100%
6. **年化收益率** = (市值/总成本)^(365.25/持有天数) - 1

## 注意事项

- 基金净值来自 `scripts/crawler.py` 爬取的最新数据
- 交易表写入后，立即更新持仓表，保证数据一致性
- 字段 ID 变更后需重新运行 `python scripts/feishu_config.py`

## 本功能常见错误

| 错误 | 原因 | 恢复 |
|------|------|------|
| 卖出收益/收益率方向错误 | 字段未按规范（正=盈/负=亏） | 卖出份额/金额/成本用负数，收益/收益率用正数 |
