# 定投交易记录更新

将定投截图中的新条目合并到交易表中最新的定投记录，并同步持仓表。

## 触发场景

- 用户说「更新定投交易」「定投截图」「发一张定投记录」「合并定投」

## 工作流

```
用户截图 ──► mineru2md 解析 ──► 识别标的代码 + 日期范围 + 每笔金额/份额
                                    │
                    ┌────────────────▼────────────────┐
                    │  2. 从交易表查询该标的最新的定投记录 │
                    └────────────────┬────────────────┘
                                    │
                    ┌────────────────▼────────────────┐
                    │  3. 筛选截图日期严格大于原记录日期的条目 │
                    └────────────────┬────────────────┘
                                    │
                    ┌────────────────▼────────────────┐
                    │  4. 合并计算（Σ金额/份额）        │
                    └────────────────┬────────────────┘
                                    │
                    ┌────────────────▼────────────────┐
                    │  5. 更新交易表记录（record-upsert）│
                    └────────────────┬────────────────┘
                                    │
                    ┌────────────────▼────────────────┐
                    │  6. 执行 trade_calc.py --resync-holdings │
                    └────────────────────────────────┘
```

## Step 1: 解析截图

```bash
mineru2md --file {image_path} --output /tmp/
```

提取每笔定投的**日期、净值/价格、金额/份额**。

**金额/份额换算**：截图给出"金额"时，份额 = 金额 ÷ 净值（保留2位小数）。

## Step 2: 查询交易表

```bash
lark-cli base +record-list \
  --base-token FlZObdBVNawsG0s9GhHch2xDnAc \
  --table-id tblkzlJG97qsMFfK \
  --limit 200 \
  --offset 0 \
  --format json
```

**注意**：需循环分页（offset += 200）直到 `has_more=false`，268条记录需查完。

筛选：
- `代码 == 目标代码`
- `方向 == ["定投"]`
- 取**交易日期最晚**的一条记录 → `record_id`

## Step 3: 筛选新增条目（核心规则）

**只取交易日期严格晚于（大于）交易表最新定投记录交易日期的条目。**

- 原记录日期 = 2026-05-29
- 截图日期 2026-05-29 → ❌ 不纳入（等于，不晚于）
- 截图日期 2026-06-01 及之后 → ✅ 纳入

## Step 4: 合并计算

```
新增金额合计 = sum(每笔新增金额)
新增份额合计 = sum(每笔新增份额) = sum(金额_i / 净值_i)

新金额 = 旧金额 + 新增金额合计
新份额 = 旧份额 + 新增份额合计（保留2位小数）
新日期 = 截图中最晚的交易日期（格式：YYYY-MM-DD 00:00:00）
新成本 = 新金额（成本 = 金额，定投不打折）
```

## 重复记录检查（重要）

**触发时机**：每次处理截图前后，都要先做一次全量重复检查并报告给用户。

**重复判定规则**：同一标的（代码相同）+ 同一方向 + 金额相同 + 份额相同，即视为可能重复，不论文档日期是否一致。

**注意**：分红记录以除权日为准，若查到两条金额/份额/方向都相同的分红，先用 `dividend_update.py --check-only` 确认哪个日期正确，再决定删除哪条。

## Step 5: 更新交易表

```bash
lark-cli base +record-upsert \
  --base-token FlZObdBVNawsG0s9GhHch2xDnAc \
  --table-id tblkzlJG97qsMFfK \
  --record-id {record_id} \
  --json '{"金额":{new_amount},"份额":{new_shares},"交易日期":"{new_date} 00:00:00","成本":{new_amount}}'
```

## Step 6: 同步持仓表

```bash
python3 scripts/trade_calc.py --resync-holdings
```

## 计算示例

旧记录：金额 1,000，份额 802.90，日期 2026-05-29
截图新增（2026-06-01 ~ 2026-06-04，共4笔，每笔100元）：

| 日期 | 净值 | 份额 = 100/净值 |
|------|------|----------------|
| 2026-06-01 | 1.2340 | 81.04 |
| 2026-06-02 | 1.2334 | 81.08 |
| 2026-06-03 | 1.2295 | 81.33 |
| 2026-06-04 | 1.2111 | 82.57 |
| **合计** | | **326.02** |

```
新增金额合计 = 400
新增份额合计 = 326.02

新金额 = 1000 + 400 = 1400
新份额 = 802.90 + 326.02 = 1128.92
新日期 = 2026-06-04
新成本 = 1400
```

## 本功能常见错误

| 错误 | 原因 | 恢复 |
|------|------|------|
| 重复记录未发现 | 只按 (代码+日期+方向) 判定 | 新规则：代码+方向+金额+份额相同即重复 |
